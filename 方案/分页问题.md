# 分页

## 常见问题
- 有总页数的需求
- 数据变化造成上下页重叠
- 列表返回的数组长度与每页数量不一致
- 根据请求参数返回格式
  - count,返回总数
  - cursor,根据游标查询
- 缓存
  - 详情根据id缓存对象
  - 列表只缓存id,分页使用游标.需拆分情况如下
    - 单key多于百万数据
    - 读写频繁(秒级多次发布下线不适合)
    - 排序规则复杂(多字段权重)
    - 极热列表(QPS极高)

## 规范
- 支持 page/page_size 和 cursor/limit 分页,默认是 cursor 方案,query 里没有 cursor 有 page 则选择 page 方案,但优先选 cursor
- cursor包含分页内容的 base64 或 hash 处理后的字符串,有 id 和时间还有排序等信息
- 请求的query可以携带控制信息(with=user,counter).返回的 body 里可能有其他附带的额外的特定业务数据
- 服务器返回状态码 200 表示请求已接收处理
- success 表示操作是否成功
- code 字段代表业务错误码
- message 字段表示错误信息
- data 可能有. list 表示是返回的列表数组,info 表示返回的详情对象,请求 query 有 with-total 返回 total 字段,has_more 和 next_cursor 表示下一页

```typescript
type Wrapper {
  code: number;
  success: boolean;
  message: string;
  data: object | {
    list: object[]
    info: object,
    has_more?: boolean; //  默认可以省略,特殊情况,过滤删除数据,导致返回列表和 page_size 不一致,可以加这个字段
    total?: number;
    next_cursor?: string;
  }
}
```
